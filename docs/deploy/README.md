## 三分鐘理解這套系統怎麼跑

```
                 使用者瀏覽器
                      │  http://校內主機IP
              ┌───────▼────────┐
              │  web (Caddy)   │  靜態前端 + 反向代理 /api
              └───────┬────────┘
                      │
        ┌─────────────▼──────────────┐
        │  api (FastAPI)             │  登入、資料維護、手動排課、報表
        └───┬──────────────────┬─────┘
            │                  │
     ┌──────▼──────┐    ┌──────▼───────┐
     │ postgres    │    │ redis        │  佇列 + session 時效
     │ (你的資料)  │    └──────┬───────┘
     └─────────────┘           │
            ▲            ┌──────▼───────┐
            └────────────┤ worker       │  自動排課、寄信、PDF 匯出、每日備份
                         └──────────────┘
```

- **你的資料只在 `postgres`**(存於 Docker volume `pgdata`),備份檔在 volume `backups`。只要這兩者在,系統可隨時重建。
- **worker** 負責耗時工作(排課引擎、寄信、PDF/PNG 匯出、每日自動備份),與 api 分離,排課時不卡畫面。
- 全部跑在**一台主機**,不需要外網、不需要雲端帳號,校內網路即可使用。

## 兩種部署方式

同一份 `docker-compose.yml` 支援兩種做法:

1. **拉取官方預建映像**(推薦,免建置、升級快):只要 `docker-compose.yml` + `.env` 兩個檔案,`docker compose pull && docker compose up -d`。
2. **從原始碼自行建置**(想改程式、或內網無法連 GHCR 時):`git clone` 本專案後 `docker compose up -d`,首次會在本機建置映像(數分鐘)。

詳見 [安裝指南](install.md)。

## 名詞對照

| 文件用詞 | 意思 |
|---|---|
| 主機 / host | 跑 Docker 的那台電腦(校內伺服器、桌機、NAS 皆可) |
| 容器 container | 一個服務的執行實體,如上圖五個 |
| volume | Docker 管理的資料儲存區,容器刪掉也不會消失 |
| `.env` | 你的設定檔(密碼、校名、埠號),**含機密,不要上傳到任何地方** |
